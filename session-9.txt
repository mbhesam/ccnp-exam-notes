udld (unidirectional link detector) ---> can detect when one fiber optic link
has failure on send or recieve thread (in this situation because we dont have 
recieve or send traffic stp think should bring up alternative link in topology) 
udld check fiber link every 15 sec.
udld modes ---> normal ---> just send snmp message but didnt change link status 
	   ---> aggressive ---> make link down 
#udld enable ---> enable udld on all links
#udld enable aggressive
-if)#udld port 
-if)#udld port aggressive 
when one link goes down because of udld aggressive mode you have to set
this command : 
-if)#udld reset 
-if)#udld recovery 15 ---> 15 sec after goes up 
#show udld int e 0/0
#show udld neighbours
#errdisable recovery cause bpduguard 
#errdisable recovery interval 15 ---> set recovery 
stp ---> IEEE 802.1D
rstp ---> IEEE 802.1W
mstp ---> IEEE 802.1S
imagine you have 100 vlans so in pvst or rstp in every 2 sec 100 bpdu transfer 
so IEEE decide to release mst. in mst we can have instance and assign some of 
vlans to  each instance so less and effective bpdu will transfer.
in fact every 2 sec just 1 bpdu will send even we have lots of instances.
all of stp instance information keep in 1 bpdu.
